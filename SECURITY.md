# 安全策略（Security）

## 支持范围

本仓库为 **GlueKit Workbench** VS Code / Cursor 扩展。我们关注会影响用户的**机密性、完整性与可用性**的问题，例如：

- 在未授权情况下读取或篡改**工作区外**的敏感路径  
- Webview 与扩展宿主之间的**消息注入**导致任意命令执行  
- **依赖供应链**中可确认的恶意行为（请一并说明版本与锁定文件）

常规功能请求、UI 建议、非安全缺陷请走 [GitHub Issues](https://github.com/gluekit/v7-cursor/issues)（若你使用独立仓库，请改为该仓库的 Issues 链接）。

## 如何报告漏洞

**首选**：在 **GitHub 本仓库**页面使用 **Security → Report a vulnerability**（私有安全建议），避免公开细节。

若仓库未开启该功能或你无法使用，请通过你在 **README 中公示的联系方式**联系维护者。**请勿**在公开 Issue 中粘贴完整可利用的 PoC；可先只描述影响范围与版本。

建议在报告中包含：

- 影响版本或 commit  
- 复现步骤与最小示例（若有）  
- 潜在影响与你的初步判断  

## 处理说明

- 我们会在合理时间内**确认收到**并评估严重性（通常为数日，视维护资源而定）。  
- 修复就绪前请**不要**公开可利用细节；我们会在发布修复版本后协调**公开披露**（如 GitHub Security Advisory）。  
- 本扩展以 **MIT** 许可提供，**按原样（AS IS）**提供，不承担法定担保义务；我们仍会认真对待有效报告。

## 非漏洞类安全建议

若你担心「扩展是否会外传代码」等：**本扩展不向 GlueKit 运营方自动上传工作区内容**；行为以开源 `src/` 为准，建议在安装前自行审查或只从 **官方发布渠道 / 可信 VSIX** 安装。
